BUDGEIT
======

BudgeIT è una web app di gestione finanziaria personale, progettata come prodotto
consumer premium, local-first, senza backend e senza framework.
L’obiettivo è offrire un’esperienza pulita, coerente e affidabile, con un’architettura
semplice ma solida.

--------------------------------------------------
PANORAMICA ARCHITETTURA
--------------------------------------------------

- Vanilla JavaScript (nessun framework)
- Router custom minimale
- Stato runtime separato dallo storage persistente
- UI modulare e riutilizzabile
- LocalStorage come unica fonte dati
- Supporto PWA (manifest + service worker)

Il progetto segue un modello:
View-driven + App Controller + State + Storage

--------------------------------------------------
DESCRIZIONE FILE E CARTELLE
--------------------------------------------------

ROOT
----

index.html
- Entry point dell’app
- Contiene solo il markup base
- Nessuna logica applicativa
- Include CSS e JS principali

manifest.json
- Configurazione PWA
- Nome app, icone, colori tema
- Modalità standalone

service-worker.js
- Gestione cache offline
- Supporto PWA
- Isolato dalla logica applicativa

--------------------------------------------------
CSS
--------------------------------------------------

/css/main.css
- Design system globale
- Colori, font, spacing
- Componenti UI (bottoni, input, card, modali, tabs)
- Gestione temi (light / dark / auto)
- Nessuna logica JS o stili inline di vista

--------------------------------------------------
JAVASCRIPT CORE
--------------------------------------------------

/js/app.js
- Controller centrale dell’app
- Bootstrap iniziale
- Coordinamento router, storage e UI
- Gestione profili (login, creazione, switch)
- Gestione tema
- Metodi UI globali (App.*)
- Utility comuni (escapeHtml, toast, alert)

/js/router.js
- Router custom minimale
- Registra le viste
- Gestisce la navigazione logica
- Chiama render() delle viste
- Gestione errori di rendering
- Non usa URL o history browser

/js/state.js
- Stato runtime temporaneo
- Dati non persistenti
- Reset ad ogni ingresso nell’app
- Nessun accesso a localStorage

/js/storage.js
- Gestione centralizzata dei dati persistenti
- Unico punto di accesso a localStorage
- Profili
- Spese
- Categorie
- Budget
- Impostazioni
- Migrazione dati legacy
- Versionamento struttura dati

--------------------------------------------------
VIEWS
--------------------------------------------------

/js/views/home.js
- Dashboard iniziale post-login
- Riepilogo e accessi rapidi
- Stato generale del profilo

/js/views/expenses.js
- Gestione spese
- Inserimento nuove spese
- Filtri (mese, categoria, ordinamento)
- Lista spese
- Eliminazione spese
- Vista altamente interattiva

/js/views/budget.js
- Gestione categorie
- Creazione e rimozione categorie
- Impostazione budget mensili
- Visualizzazione progresso
- Warning visivi (80% / 100%)

/js/views/stats.js
- Statistiche e analisi
- Aggregazioni dati
- Breakdown per periodo e categoria
- Vista read-only (nessuna scrittura dati)

/js/views/onboarding.js
- Flusso di avvio dell’app
- Primo utilizzo
- Login profili esistenti
- Creazione nuovo profilo
- Gestione PIN
- Unica vista accessibile senza profilo attivo

--------------------------------------------------
UI COMPONENTS
--------------------------------------------------

/js/ui/alert-modal.js
- Modale di conferma e input
- Conferme distruttive
- Input controllati
- Messaggi bloccanti

/js/ui/settings-modal.js
- Modale impostazioni
- Gestione tema
- Gestione profilo
- Cambio profilo

/js/ui/toast.js
- Notifiche non bloccanti
- Feedback azioni
- Success / error leggeri

--------------------------------------------------
ICONE
